this project is a library, called HT, for accessing simple.fleetgrid.com's database commands. starting as a library for HTML projects the base API is programmed in JS. however, because messages are transmitted between FG and HT over the standard "ws://" WebSocket protocol ... there is nothing to stop anyone from using this --> WS backend with other languages (Swift, Java, C#, etc...).

the infrastructure around FLEETGRID is built for supporting massive amounts of vehicles. at full speed ahead scale! FG is capable of handling traffic, geospacial, routing, and field service. it doesn't leave out the solo-driver or small-to-medium size teams either. it does all this exceptionally well.

with FG as a bases for HT's simple database library a world of possibilities beigns to unfold. checkout the game // tyu67.com which showcases HT's ability to support objects ... in a sort of "faster than real-time" way and it's --> quite impressive.

application programming interface:

  • ht.beep()
  • ht.turn()
  • ht.neutral.get()
  • ht.neutral.set()
  • ht.checkpoint.start()
  • ht.checkpoint.ack()
  • ht.drive.get()
  • ht.drive.set()
